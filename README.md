# FPGA-Based-Elder-Fall-Detection-System-using-Artix--7-FPGA
Elder Fall Detection System on FPGAAn ultra-low latency, edge-based hardware solution for real-time elderly fall detection. This system utilizes a digital 3-axis MEMS accelerometer interfaced with an Artix-7 FPGA development board using a custom-designed Serial Peripheral Interface (SPI) master architecture written completely in Verilog HDL. By executing execution and decision-making logic directly at the hardware layer, the system minimizes notification latency and ensures complete privacy preservation compared to traditional camera-based or cloud-dependent monitoring systems.  🚀 Key FeaturesHardware-Driven Low Latency: The processing, synchronization, and decision routines operate completely within milliseconds, mitigating system lag common in software or OS-dependent setups.  Custom SPI Master Core: Features a dedicated Verilog SPI master configured for SPI Mode 0 (CPOL=0, CPHA=0) supporting high-speed burst reads for immediate motion processing.  Threshold-Based State Machine: Incorporates a modular Finite State Machine (FSM) that intelligently maps out real-time acceleration data to isolate true fall signatures from everyday activities.  Privacy-Preserving & Safe: Eliminates the privacy risks associated with optical or visual tracking tools, making it ideal for continuous home use.  Instant Alert Trigger: Immediate generation of audible or visual alert lines (Buzzer/LED lines) upon continuous threshold evaluation.  🛠 Hardware Architecture & ToolsHardware EquipmentFPGA Development Board: Xilinx Artix-7 Development Board (Nexys A7 / XC7A100T)   Sensor Module: Analog Devices ADXL345 3-Axis Digital Accelerometer   Peripherals: 5V Active Buzzer, Onboard LEDs, Breadboard, and Jumper Wires   Software ToolchainDevelopment Environment: Xilinx Vivado Design Suite   Hardware Description Language: Verilog HDL   🧠 Working Principle & FSM Flow[ ADXL345 Sensor ] ===(SPI Protocol)===> [ Artix-7 FPGA Core ] ===> [ Alert System ]

Initialization: Upon power-up, the SPI Master initializes communication, writing to the sensor's internal POWER_CTL register (0x2D) to trigger Measurement Mode.  Burst Data Acquisition: The master triggers automatic address auto-increment loops starting at register 0x0E to collect continuous 16-bit signed acceleration vectors ($X, Y, Z$) at a stable rate (~100 Hz sampling).  Fall Logic Evaluation: The top-level design isolates and sign-extends the vertical axis data component ($Z$-axis). If the $Z$-axis vector drops below a calibrated free-fall/low-acceleration threshold consistently for 3 seconds, it confirms a permanent state change (e.g., lying down on the floor post-impact) and trips the alarm driver line.  Instant Self-Reset: If data registers a recovery value within that period, the internal counters drop to zero instantly to prevent false alarms from everyday movements.
